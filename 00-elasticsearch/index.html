Elasticsearch 简介

Elasticsearch 是什么？

ElasticSearch是一个基于Lucene的开源分布式搜索和分析引擎服务器。它的特点有：分布式，零配置，自动发现，索引自动分片，索引副本机制，restful风格接口，多数据源，自动搜索负载等。它提供了一个分布式多用户能力的全文搜索引擎，基于RESTful web接口。Elasticsearch是用Java开发的，并作为Apache许可条款下的开放源码发布，是第二流行的企业搜索引擎。设计用于云计算中，能够达到实时搜索，稳定，可靠，快速，安装使用方便。并且在elasticsearch集群中，所有节点的数据是均等的。

Elasticsearch 能做什么？

ElasticSearch 是一款非常强大的开源搜索及分析引擎，可以帮助你从海量数据中，快速找到你想要得到的相关信息，列如,当你在github上面搜索，Elasticsearch不仅可以帮你找到相关的代码仓库，还可以帮助你实现代码级别的搜索与高亮显示，当你购物的时候，Elasticsearch可以帮助你推荐相关的商品，当你打车回家的时候，Elasticsearch可以通过定位附近的乘客和司机，帮助平台优化调度。

除了搜索，结合kibana，logstash、beats、elastic stack 还被广泛的运用在大数据近实时分析领域，包括日志分析，指标监控，信息安全等多个领域，它可以帮助你探索海量结构化，非结构化的数据，按需创建可视化报表，对监控数据设置报警阈值，甚至可以通过机器学习技术，自动识别异常状况。

为什么学习Elasticsearch

在DBRanking 2019年5月的数据库评测中，Elasticsearch在Search Engine分类的名列第一，2019年Dice技术薪资报告显示，今年美国IT薪资最高技能排名中，Elasticsearch技术排名第六。 国内现在有大量的公司都在使用Elasticsearch，包括滴滴，今日头条，饿了吗，360，小米，vivo，携程等诸多知名公司，今天，elasticsearch在全球的下载量超过3.5亿次，Google，亚马逊，微软，和阿里巴巴等云计算公司都在云平台提供了elasticsearch的托管服务。 目前有越来越多的人都希望学习Elasticsearch，所以elastic公司也推出来elastic工程师的认证考试，这是一门含金量很高的技术认证。在大数据时代，掌握近视时搜索和分析能力，才能掌握核心竞争力，洞见未来。

Elasticsearch 发展历史

Elasticsearch 背后的公司Elastic inc已经在在上市了，在2018年美国股市不好的情况下，这家上市公司在开盘当天涨幅度94%，当前市值50亿美金，这一切都归属与这家开源软件公司背后的数据，Elasticsearch软件下载量，超3.5亿次，10万+的社区成员，7200+订阅用户，分布在100多个国家，各大云服务商都提供了Elasticsearch的云服务(elastic,amazon,阿里巴巴,腾讯)

Elasticsearch 起源于一个基于Java语言开发的搜索引擎库类Lucene，Lucene创建1999年，2005年成为Apache开源项目，Lucene具有高性能，易扩展的优点，但是有优点就有缺点，Lucene只能基于Java语言开发，类库的接口学习曲线陡峭，原生并不支持水平扩展。2004年Shay Banon基于Lucene开发了Compass，2010年Shay Banon重写了Copass，并取名Elasticsearch(支持分布式，可水平扩张，降低全文检索的学习曲线，可以被任何编程语言调用),集群节点可以从单个扩展至数百个节点,高可用&水平扩展,并且支持给节点赋予不通的角色类型。并且Elasticsearch提供了多种编程语言的类库(java,.net,python,php,golang等),还可以通过RESTful API方式

Elasticsearch 2010年第一次发布，2012年成立公司，成立6个月，160万下载，首轮募到1000万美元风险投资。这样也是由于elastic公司把产品放在第一位，对外宣传”不要求你必须是一位数据科学家才能把它用好”

Elasticsearch 版本与升级 - 04:2010年2月第一次发版 - 1.0:2014年1月 - 2.0:2015年10月 - 5.0:2016年10月 - 6.0:2017年10月 - 7.0:2019年4月

可以关注elastic官网查看更新的特性

Elasticsearch 家族成员

Elastic Seack 生态圈

Logstash

Logstash是开源的服务器端数据处理管道，它可以对你的数据进行收集、过滤、分析，支持大量的数据获取方法，支持从不同的数据源采集数据，转换数据，并将其发送到不用的存储库中，Logstash诞生于2009年，最初用来做日志的采集与处理，2013年被elasticsearch收购。logstash比较好的特性，列如根据ip破译出地理坐标，或者将数据进行脱敏等，logstash提供了200多个插件(日志，数据库等),而且logstash的数据是支持加密传输，通过持久化队列来保证至少将运行的事件送达一次。

Kibana

Kibana是一个基于浏览器页面的Elasticsearch前端展示工具，也是一个开源和免费的工具，Kibana可以为 Logstash 和 ElasticSearch 提供的日志分析友好的 Web 界面，可以帮助您汇总、分析和搜索重要数据日志。

Beats

Logstash采集日志会费很大的性能问题，然后elastic提供了轻量级的数据采集器，这些采集器是用于go语言开发的，所以运行也比较方便，(filebeat,packtbeat,winlogbet等)

X-Pack

xpack时es的商业版的套件，拥有很多的安全特性，6.8和7.1开始一些安全功能免费，但是针对于一下企业用的功能还是不免费，列如，ldap&ad对接等

Elasticsearch 应用场景

基于Easticsearch架构

Elasticsearch应用场景用于(网站搜索，垂直搜索，代码搜索，日志管理分析，安全指标监控，应用性能监控，web抓取舆情分析) - 日志系统:比如运维是一个医生，系统是一个病人，那么日志就是系统对自己的陈述，日志系统还可以发现一些恶意攻击，恶意刷单，注册，密码猜测等，我们目前公司基于elasticsearch做了基于日志为数据的应用风控系统。而且通过kibana上面对日志进行机器学习，还可以针对一些错误发出告警。 - 与数据库集成:可以把mysql的数据实时同步到ES，实现低延迟的检索到ES中的数据或者进行其它数据分析处理

基于Easticsearch案例

国外 - （1）维基百科，类似百度百科，牙膏，牙膏的维基百科，全文检索，高亮，搜索推荐 - （2）The Guardian（国外新闻网站），类似搜狐新闻，用户行为日志（点击，浏览，收藏，评论）+社交网络数据（对某某新闻的相关看法），数据分析，给到每篇新闻文章的作者，让他知道他的文章的公众反馈（好，坏，热门，垃圾，鄙视，崇拜） - （3）Stack Overflow（国外的程序异常讨论论坛），IT问题，程序的报错，提交上去，有人会跟你讨论和回答，全文检索，搜索相关问题和答案，程序报错了，就会将报错信息粘贴到里面去，搜索有没有对应的答案 - （4）GitHub（开源代码管理），搜索上千亿行代码 - （5）电商网站，检索商品 - （6）日志数据分析，logstash采集日志，ES进行复杂的数据分析（ELK技术，elasticsearch+logstash+kibana） - （7）商品价格监控网站，用户设定某商品的价格阈值，当低于该阈值的时候，发送通知消息给用户，比如说订阅牙膏的监控，如果高露洁牙膏的家庭套装低于50块钱，就通知我，我就去买 - （8）BI系统，商业智能，Business Intelligence。比如说有个大型商场集团，BI，分析一下某某区域最近3年的用户消费金额的趋势以及用户群体的组成构成，产出相关的数张报表，**区，最近3年，每年消费金额呈现100%的增长，而且用户群体85%是高级白领，开一个新商场。ES执行数据分析和挖掘，Kibana进行数据可视化

国内 - （9）国内：站内搜索（电商，招聘，门户，等等），IT系统搜索（OA，CRM，ERP，等等），数据分析（ES热门的一个使用场景）
